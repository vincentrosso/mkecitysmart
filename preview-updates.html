🛡️ Crashlytics Fix — Network Errors

No UI change — backend fix that stops the crash emails

What was happening
When a user's phone lost internet (subway, dead zone, airplane mode), any HTTP call crashed the app and sent an error to Firebase Crashlytics:

ClientException with SocketException: Network is unreachable
What we fixed
Created SafeHttpClient wrapper applied to all 9 HTTP services. Network errors are now caught gracefully — the app shows "unavailable" states instead of crashing.
